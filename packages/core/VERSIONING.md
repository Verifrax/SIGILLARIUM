# Versioning Policy — @sigillarium/core

- Follows Semantic Versioning (SemVer).
- `MAJOR`: breaking API or schema change.
- `MINOR`: backward-compatible feature additions.
- `PATCH`: backward-compatible fixes.
- Schema rules:
  - v1 canonicalization rules are frozen.
  - Any canonicalization or schema rule change requires new schema version and MAJOR bump.
